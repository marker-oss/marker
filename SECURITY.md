# Политика безопасности

## Сообщить об уязвимости

- **Незначительные проблемы** — открывайте GitHub Issue.
- **Серьёзные уязвимости** (утечка данных, обход авторизации и т.п.) — свяжитесь напрямую с мейнтейнером по электронной почте, не публикуя детали в открытом трекере.

## Модель развёртывания (важно)

Веб-сервер **не имеет аутентификации**. `wrap-cors` разрешает все источники (`#".*"`); никакого middleware для логина или базовой HTTP-аутентификации нет.

Следующие POST-эндпоинты меняют состояние или запускают фоновые задачи:

| Эндпоинт | Действие |
|---|---|
| `POST /plan` | сохраняет план закупки (HTMX-страница) |
| `POST /api/sync/start` | запускает синхронизацию данных с маркетплейсом |
| `POST /api/sync/run` | строит план задач и запускает параллельный синк |
| `POST /api/sync/tasks/*/retry` | повторяет упавшую задачу синхронизации |
| `POST /api/sync/stop` | останавливает текущую синхронизацию |
| `POST /api/sync/rematerialize` | перепрогоняет материализацию из сырых данных |
| `POST /api/sync/schedule` | обновляет расписание автосинхронизации |
| `POST /api/cost-prices/upload` | загружает CSV себестоимостей из 1С |
| `POST /api/cost-prices/preview` | предпросмотр CSV себестоимостей |
| `POST /api/v1/marker/what-if-recalc` | пересчитывает what-if сценарий по артикулу |

**Запускайте сервер только в доверенной локальной сети.**
Не выставляйте его в публичный интернет без обратного прокси (nginx/Caddy и т.п.)
с аутентификацией перед ним.

Подробнее о вариантах развёртывания: [docs/deploy.md](docs/deploy.md).

## Секреты

API-токены маркетплейсов хранятся в `.env` и `config.edn`.
Оба файла перечислены в `.gitignore` — не коммитьте их ни при каких обстоятельствах.
